🤖 mono·pricehelper — AI-прайсер для monoбазар

Інтелектуальний агент для визначення справедливої ринкової ціни товару на б/у маркетплейсі. Приймає опис та фото від продавця, аналізує реальні ринкові дані та повертає рекомендовану ціну, стратегії продажу та детальне пояснення за < 5 секунд.

⚡ Як це працює

Продавець вводить опис + завантажує фото, після чого паралельно запускаються два процеси:

ML-регресія (Ridge + HistGradientBoosting + FAISS) — відпрацьовує за ~0.3с.

Gemini 2.5 Flash Lite (Мультимодальний аналіз) — відпрацьовує за ~2-4с.

Якщо Gemini встигає у таймаут — повертається детальна відповідь з урахуванням стану на фото. Якщо ні (або зник інтернет) — миттєво повертається ML fallback-відповідь на основі математики. Загальний час відповіді користувачу завжди < 5с.

✨ Унікальні фічі (Edge Cases Handling)

Наша система стійка до типових помилок користувачів:

🎯 Category Auto-Correction: Якщо продавець помилився і вибрав категорію "Шини" для "iPhone 13 Pro", семантичний KNN-класифікатор перехопить вектор, зрозуміє помилку і примусово змінить категорію на "Смартфони" ще до запуску ціноутворення.

🔍 True Query Expansion: Розпізнавання "лінивого сленгу". Якщо юзер пише "13 про", система на льоту розширює запит до "13 pro apple iphone" для ідеального пошуку у FAISS, залишаючи саму базу "чистою".

🛡️ Antique Immunity: Вбудований фільтр "божевільного продавця" відсікає космічні ціни під час навчання. Але якщо модель бачить ознаки антикваріату чи колекційки (Regex детектори), фільтр знімається, дозволяючи ціні зростати пропорційно цінності.

🧮 Multi-modal Math: Gemini інструктована рахувати об'єкти на фото (напр., 4 стільця). Якщо ML віддає ціну за 1 шт., ШІ самостійно виконає перемноження і видасть ціну за весь комплект.

🧠 ML Pipeline (Stacking)

Ми використовуємо 3-рівневу архітектуру для роботи з екстремально різнорідними даними (Train MAPE: ~28-30%):

Ridge + TF-IDF — ловить назви брендів, моделей, специфікацій з тексту у логарифмічному просторі.

FAISS Semantic Search — векторний пошук. Знаходить медіану цін 5-50 семантично найближчих товарів (KNN Feature). Працює як "очі" системи.

HistGradientBoosting — "мозок". Ансамбль дерев рішень об'єднує результати Ridge, підказки від FAISS та структурні фічі (бренд, відсоток батареї, довжина опису, терміновість) для фінального передбачення.

Навчання відбувається на таргеті log1p(sold_price) — ми орієнтуємося на реальні ціни угод, а не фантазії продавців.

📊 Стратегії продажу

Розраховуються на основі реальних угод та дисконтів у конкретній категорії:

FAST — швидкий продаж за 1-3 дні (ціна зі знижкою)

BALANCED — оптимальний варіант

MAX_PROFIT — максимальний прибуток (рекомендована ціна + преміальна націнка 20-25%)

🚀 Встановлення та Запуск

Вимоги: Python 3.10+, Google Gemini API key.

# 1. Клонування репозиторію
git clone [https://github.com/ВАШ_ЛОГІН/mono_skelar.git](https://github.com/ВАШ_ЛОГІН/mono_skelar.git)
cd mono_skelar

# 2. Створення віртуального середовища
python -m venv .venv
source .venv/bin/activate  # Для Windows: .venv\Scripts\activate

# 3. Встановлення залежностей
pip install -r requirements.txt

# 4. Налаштування API-ключів
echo "GEMINI_API_KEY=your_api_key_here" > .env


Переконайтеся, що файли даних лежать у папці data/.

Запуск додатку

# Веб-інтерфейс (Flask)
python server.py
# Відкрити: http://localhost:8080

# CLI-демо (7 складних кейсів для тестування)
python main.py


📡 API Reference

POST /api/price
Content-Type: multipart/form-data

Поля запиту:

description (string, обов'язкове) — опис товару.

category_id (integer, опційне) — ID категорії.

images (file[], опційне) — фото товару.
